Question(1): Discuss the scope of 'var', 'let', and 'const'.

** 'var', 'let', 'const' is a keyword used to declare variables.
** It has function-level scope which means they are accessible throughout the entire function in which they are defined. if 'var' declared outside of any function it become global variable, it can be accessible from anywhere in your code. It also hoisted. It can be re-declared and updated.
** 'let' and 'const' have block-level scope, which means they are accessible only within the block where the are defined. 'let' are mutable which means you can reassign their values after they have been declared.
And 'const' are immutable, which means their values cannot be reassigned after they are initially assigned. 'const' are not hoisted.
In simple word 'let' can be updated but not re-declared, while 'const' cannot be updated or re-declared.

Question(2): Tell us the use cases of null and undefined

1. 'undefined':
'undefined' represents the absence of a value or the lack of an assigned value to a variable. It is a primitive data type in javaScript.
Some Common use cases of 'undefined':
i. Variable that have been declared but not assigned a value.
ii. function parameters that have not been provide with an argument.
ii. The return value of a function that doesn't explicitly return anything.

2. 'null':
'null' is a deliberate assignment of an empty or non-existent value to a variable. And it also a primitive data type in JavaScript.
Some Common use cases of 'null':
i. Often used when want to reset or clear the value of a variable.
ii. Explicitly indicating the absence of an object or value.
ii. It is sometimes used as a placeholder value in objects.

Question(3): What do you mean by REST API?

REST API:
A REST API (Representational State Transfer Application Programming Interface) is a standardized way for software Application in JavaScript that allows JavaScript applications to send HTTP request to interact with remote servers or services, typically to retrieve or update data. It follows the principles of REST, using Standard HTTP methods like GET, POST, PUT, and DELETE to perform actions on resources. This enables JavaScript applications to communicate with external data sources and service over the internet.
